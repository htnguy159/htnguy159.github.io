Project Server Design: Budget Web App

Full Stack Web App: Budgeting App

For my current project, I am developing a Budgeting Web App with my teammate, Phuong-Anh.

Our goal is to keep track of user expenses and provide monthly reports to show spending habits.

This week we focused on the Server Design that this article showcases below.

REST API



Server Logic

  • Main View (Sign Up View)

    • /api/signUp

    • Create session
    • Server require users to fill out all required user inputs (Sign up form)
    • Prevent pre-registered emails from re-registration
    • After signing up, user is redirected to log in view

  • Log In View

    • /api/logIn

    • Create session
    • User authentication
    • Server require users to fill out all required user inputs (Log in form)
    • After logging in, user is redirected to general view

  • General View

    • /api/generalView

    • When user inputs data, the chart and graphs update
    • Server require users to fill out all required user inputs (Add expense form)
    • User stays logged in until they press log out

Server Fetches

  • /api/generalView (List, Pie Chart, and Bar Graph): GET, POST for ChartJS to create/update charts
  • /api/signUp & /api/logIn: Express fetches GET request of session id

Tooling & Libs

  • EJS, Bcrypt, Passport, ChartJS, Express
