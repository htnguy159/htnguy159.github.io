Project Client Design: Budget Web App

Full Stack Web App: Budgeting App

For my current project, I am developing a Budgeting Web App with my teammate, Phuong-Anh.

Our goal is to keep track of user expenses and provide monthly reports to show spending habits.

This week we focused on the Client Design that this article showcases below.

Mockups

  • Main View/Sign up view
    • Email, name, password, sign up button
  • Log in View
    • Email, password, log in button
  • General View
    • User Input:
      • Add Income
      • Add Expense
    • Expense History
    • Remaining Balance
    • Pie Chart
    • Bar Graph
    • Log Out Button

Client Logic

  • Main View (Sign Up View)

    • Starting point for application - Form for users to register an account

    • Inputs: Email, Name, Password
    • Option Sign Up: Go to logged in General View
    • Option: Already have an account? Log in: Go to Log In View

  • Log In View

    • User Authentication - Form to access user profile

    • Inputs: Email, Password
    • Option Log In: Go to logged in General View or Alert user of invalid Email/Password

  • General View

    • Header

    • Option Log Out: Go to Main View
    • Inputs: Monthly income, description, category, date, amount
    • Option 1: Submit (Add expense)

Server Fetches

  • Main View: POST and GET request for Sign Up
  • Log In View: POST and GET request for Log In
  • General View: Maybe POST for the graphs

Tooling & Libs

  • Main View: HTML, CSS, JS, JSON, EJS
  • Log In View: HTML, CSS, JS, JSON, EJS
  • General View: HTML, CSS, JS, ChartJS
