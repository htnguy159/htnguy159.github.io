Project Specifications: Budget Web App

Full Stack Web App: Budgeting App

For my current project, I am developing a Budgeting Web App with my teammate, Phuong-Anh.

Our goal is to keep track of user expenses and provide monthly reports to show spending habits.

In order to do this we need a solid plan.

In this article, I've included all of our specifications that we've decided to include in our project that you can look forward to!

  • User Stories
  • Front End Features
  • Backend Features
  • Database Features

User Stories

    Regular User

  • Sign Up
  • Log In
  • Add an Expense
  • View All Submitted Expenses
  • View Monthly Category Pie Chart
  • View Monthly Total Spending Bar Graph
  • View Remaining Budget Balance
  • Log Out

    • Guest User

  • Sign Up
  • Log In
  • Add an Expense
  • View All Submitted Expenses
  • View Monthly Category Pie Chart
  • View Monthly Total Spending Bar Graph
  • View Remaining Budget Balance
  • Log Out

Frontend Features

    Index

  • Sign Up View - User Input:
    • Form to sign up
      • Email
      • Name
      • Password
    • Submit Button for Sign Up
    • Link to login under form (for established users)
  • Login View - User Input:
    • Form to Login
      • Email
      • Password
    • Submit Button for Sign Up

    General View (Once Logged In)

  • User Input:
    • Adding an Expense
      • Title
      • Category (Drop down menu)
      • Date
      • Amount
      • Submit Button for Add Expense
  • Chart/List detailing the expenses for the current month
  • Pie chart detailing the amount for each expense category of the current month (EX: Savings: 15%, Personal 50%, Fees: 5%, Transportation 15%, Education: 15%)
  • Bar graph of previous months detailing the total amount spent in a month
  • Display of remaining budget balance: Monthly income - total amount spent
  • Log out option at top right screen

    • Backend Features

  • Establish User Sign Up
  • Authenticate User
  • Updates chart/list
    • Takes the user’s input and converts it to show in the chart as a list
    • Notify user that new expense has been successfully submitted
  • Calculates percentages of each category with the given user input
  • Updates pie chart
    • Converts user’s input to give a visual of how much each category is in relation to the other categories (based on the percentages)
  • Calculates the remaining budget balance
  • Calculates the total amount spent for each month
  • Updates bar graph
    • Uses the calculated total amount spent monthly and displays a bar graph to see the difference from month to month
  • Allow logged in users to log out

    • Database Features

  • User data
    • Data from previous sessions for users who log in
    • Data from current session
    • Record of users who have accounts
      • Users’ emails
      • Users’ name
  • Dates
    • Day
    • Month
    • Year
  • Catergories
    • Savings
    • Personal
      • Food
      • Clothes
      • Entertainment
      • Medical Health
    • Fees
      • Electricity
      • Water
      • Internet
      • Phone
    • Transportation (Personal car or public transportation
    • Education
    • Other
